# 素材上传指南

本指南包装了一个命令行工具 (`byted-viral-replicator-media`)，用于管理素材。其核心设计是提供一个无状态、由调用方驱动的素材分组工具。

## 目的与适用场景

本指南的主要目的是**按用户意图进行素材分组管理**。它不提供任何智能分析或自动分组功能。

## 输入与参数

本指南的所有操作都基于对 `byted-viral-replicator-media` 工具的调用，其核心参数如下：

- **文件路径**: 要操作的本地素材文件的相对路径。
- **`--group`**: （必需）字符串，用于标识素材所属的分组。该 ID 完全由上层 Agent 或工作流在调用时指定。
- **`--metadata`**: （必需）JSON 字符串，通常包含 `channel`、`chat_id` 等元信息。脚本会利用这些信息，但指南本身不强制其内容。

## 执行流程与关键约束

Agent 在使用此指南时，应遵循以下流程：

1.  **单次上传**：
    - 调用 `add` 命令上传单个素材。
    - **必须**在对话中向用户明确反馈返回的**素材ID**和**分组ID**。
    - **必须**询问用户是否要继续向该分组上传素材。

2.  **批量上传**：
    - 由 Agent 循环调用 `add` 命令，每次传入相同的 `--group` ID。
    - 在所有上传操作完成后，**合并一次性反馈**给用户，内容应包括：
        - 所有新上传素材的 ID 列表。
        - 它们所属的统一分组 ID。
        - 询问用户是否还需继续上传。

3.  **会话结束**：
    - 为实现“无持久化”的效果，工作流或 Agent **必须**在会话/任务结束时，对本次使用过的所有 `group` ID **统一调用 `clear` 命令**。
    - 这将删除脚本在临时目录中生成的对应 CSV 文件，从而清理数据。

4.  **显式分组**：
    - 分组完全由调用方在调用时通过 `--group` 参数指定。
    - 同一分组仅表示这些素材“希望被一起消费”，不包含任何内容相似性的含义。
    - **允许重复**：同一文件可以被多次添加到同一分组中，每次都会生成一条新的素材记录。

## CLI 使用示例

所有命令均通过 `byted-viral-replicator-media <command> [arguments]` 的形式执行。

### 1. `add`：添加素材到分组

```bash
# 将 image.png 添加到 grp-001 组
byted-viral-replicator-media add ./image.png \
  --group grp-001 \
  --metadata '{"channel": "some-channel", "chat_id": "user-123"}'
```
**说明**：脚本会返回素材信息，其中包含一个唯一的素材 ID。

### 2. `list`：列出分组内的所有素材

```bash
# 查看 grp-001 组内的所有素材
byted-viral-replicator-media list \
  --group grp-001 \
  --metadata '{"channel": "some-channel", "chat_id": "user-123"}'
```
**说明**：用于在会话中获取某分组已包含的素材列表。

### 3. `clear`：清空整个分组

```bash
# 删除 grp-001 组的所有记录
byted-viral-replicator-media clear \
  --group grp-001 \
  --metadata '{"channel": "some-channel", "chat_id": "user-123"}'
```
**说明**：这是实现“无持久化”的关键。推荐在工作流结束时调用。

### 4. `remove`：从分组中移除单个素材

```bash
# 从 grp-001 组中删除指定 ID 的素材
byted-viral-replicator-media remove \
  --id <material_id> \
  --group grp-001 \
  --metadata '{"channel": "some-channel", "chat_id": "user-123"}'
```

## 注意事项

- **不要依赖内容相似度**：分组的唯一依据是调用时传入的 `--group` 参数。
- **组 ID 由上层生成**：Skill 或其包装的脚本不负责生成或管理组 ID。Agent 或上层工作流需要自行维护组 ID 的生成和传递。
- **慎用 `remove` 命令**：鉴于 `byted-viral-replicator-media` 当前的实现，推荐使用 `clear` 命令在会话结束时进行组级别的整体清理，而不是在会话中进行单个素材的移除。
