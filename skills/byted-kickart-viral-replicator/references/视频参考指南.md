# 视频参考指南

## 目的与适用场景
当用户意图进行「爆款裂变」或主动提供参考视频时，使用本指南引导用户提供并校验参考视频，最终获取可用的参考视频素材ID。

## 支持的输入方式
- 支持用户提供有效的**视频链接**。
- 支持用户**本地上传**视频文件。
- **不支持**选择 SAAS 资产库内视频。

## 视频规格要求
- **格式**：仅支持 `MP4` 或 `MOV`。
- **大小**：`≤50MB`。
- **时长**：`≤60s`。
- **分辨率**：`≥480p`。
- **比例**：支持 `9:16`, `16:9`, `3:4`, `4:3`, `1:1`。

## 交互方式与话术规范（强制遵守）
1. **引导提供视频**：
   当用户表达了爆款裂变的意图，但尚未提供参考视频时，必须按照以下话术引导：
   > 好的，我帮你创建爆款克隆项目。请先提供 参考视频：可以发我视频链接，或上传 MP4/MOV 文件（≤60s、≤50MB、≥480p）

2. **错误/超限提示**：
   如果用户上传的视频不符合规格要求（脚本返回错误），必须根据错误信息明确指出具体超限的数值，并引导用户重新提供。例如：
   > 这条视频 大小 80MB 超过 50MB 限制，请压缩或换一条再发我
   *(注意：请根据实际的超限指标（如时长、分辨率等）灵活替换上述模板中的字段)*

## 视频处理流程
用户提供视频后，必须按照以下步骤通过 `byted-viral-replicator-media` 工具上传文件，以获取合法的引用信息：

### 步骤1：获取本地文件
`byted-viral-replicator-media` 工具仅支持本地文件路径作为输入。
- **如果用户直接上传了文件**：可直接使用系统提供的文件路径。
- **如果用户提供的是视频链接**：必须先将视频下载到本地，操作规范如下：
  1. 确保下载目录存在：
     ```bash
     mkdir -p /tmp/openclaw/replicator/download
     ```
  2. 使用 `wget` 或 `curl` 下载并保存到该目录，需生成唯一文件名：
     ```bash
     # 示例：
     wget -O "/tmp/openclaw/replicator/download/video_$(date +%s%N).mp4" "<视频链接>"
     ```

### 步骤2：上传素材
阅读 `references/素材上传指南.md`，根据指南规范正确调用 `byted-viral-replicator-media` 工具，将视频上传到指定的分组中。
**注意：必须使用从主流程（如爆款裂变任务）中传递下来的统一分组ID，以确保所有相关联的素材被归类在同一个组下。**

### 步骤3：提取并保留结果
执行上传命令后，从控制台输出或返回结果中提取上传成功后的 **素材ID**。该素材ID必须妥善保留，并将作为 `ref_video` 参数传递给后续的「爆款裂变」主流程。

### 步骤4：向用户反馈
获取素材ID后，**必须**使用以下模板向用户反馈结果。
**严格要求**：必须使用普通正文格式展示，绝不可使用加粗语法（`**`），并且必须严格保留每一行的换行符！

## 📤 返回消息模板（强制遵守）

```text
✅ 参考视频上传成功 🎉
📁 素材分组：[替换为实际分组名]
🆔 素材ID：[替换为实际material_id]
📝 素材概述：[替换为实际素材信息，如视频格式、分辨率、时长等]
```
